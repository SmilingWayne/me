# Claude Code Skill：refine-markdown-to-mkdocs

先说为啥要做这个Skill。

众所周知我是一个非常喜欢做笔记整理和归纳的人，尤其是对于我非常感兴趣的书籍，在网站上磨磨蹭蹭写了快150万字的笔记，涵盖文史哲艺和我自己研究的一些方向。不过实际产出的时候会遇到一个现实的困境：

电子版书籍可以直接导出电子笔记成粗 markdown 格式，**但是这份笔记是没有额外整理过的，只是书籍或者论文、知识库文本的罗列**。这有如下几个问题

1. 其中包含了一些**信息密度极低的、杂糅的、前后关联性不大**的文本，它们可能只是我阅读过程中顺手的划线或者标记，本身并不重要；
2. 纯文本直接输入我目前工作流 (markdown -> mkdocs materials -> github workflow) 的时候太单调乏味，**缺少必要的格式高亮、卡片知识点标记等高级语法**；
3. **单一markdown文本往往基于一本书制作而来**，这也暗中限定了其中的内容是有特定风格，或者属于特定体裁的（小说、散文、历史、哲学、计算机、数学建模、人物传记或者仅仅是资料的整合），不同 markdown 文本之间所需要关注的重点不同；

> 举个例子，对于散文，可能更关注写作风格、内容、文字氛围和作者的文学特点；
>
> 对于小说，可能更关注情节特色、人物形象、时代背景；
>
> 对于历史，可能更关注整体脉络、事件意义、各方势力；
>
> 对于计算机或者我研究生做的数学建模等，可能更关注表述的准确、前后的关联、解决的问题与实际方法。

尽管对Skill了解不深，但综上所述，我可以把我需求大致可以归类为，希望有这样一个智能体：

1. 能够对于我任意给定的文本，自动判断笔记内容风格、体裁、类别，由此确定应该关注的重点；
2. 根据确定的风格重点与我定义的删改幅度，按照我指定的侧重点，对原始文本，在不进行大幅度删减、重要文本保持原样的基础上，进行语义归纳、事实梳理、前后整合、话题合并；
3. 对于整合后的文本，制作成符合我想法的 markdown 文本，支持 mkdocs materials 风格。
4. 输出总结报告、修改要点。

于是就有了这个 Skill。

## 正式版简介

`refine-markdown-to-mkdocs` 是一个为我的工作流打造的Skills，核心用于将个人零散、杂乱的 Markdown 格式读书笔记、学习笔记，通过**智能分类、语义聚类、内容重构**，精炼为符合 MkDocs Material 规范的结构化、高密度知识文本。

区别于普通格式化工具，本技能允许对内容进行**灵活合并、精简删减、轻度补充、逻辑重写**，在保留核心信息的前提下，最大化提升笔记可读性与知识密度，适配个人 MkDocs 知识库搭建与长期知识沉淀。

**核心原则是：领域自适应结构化 + 分类专属逻辑 + 可控信息保真 + 高密度知识输出**

### 支持处理范畴

覆盖**13大知识领域**的笔记智能分类与优化，包含：哲学理论、戏剧、小说、诗歌、文艺理论、散文、历史、艺术理论、传记、编程与计算机科学、数学模型与理论、跨领域整合素材、其他杂项笔记。

每个领域均配套专属**高/中/低价值内容判断规则**，精准保留核心知识点，剔除冗余噪音。

### 完整工作流程

1. **内容识别与分类**：自动读取笔记，识别书籍/章节/体裁，智能匹配知识领域并向用户确认分类
2. **语义聚类分析**：按领域规则拆分知识点，生成精炼方案，标注合并/保留/删除内容，等待用户确认
3. **结构化内容重构**：基于确认方案，将同类知识点整合为 MkDocs 风格模块，严格遵循格式规范
4. **文件自动保存**：生成精炼后文件，自动备份原文件，输出精炼方案与处理报告
5. **完成统计反馈**：展示处理数据、合并记录、删除内容摘要，全流程可追溯
